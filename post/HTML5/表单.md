# 表单

表单是网页与用户交互的核心机制，用于收集用户输入的数据（如登录信息、搜索请求、文件上传）并提交给服务器处理。

<div style="background-color: #e3f2fd; border-left: 5px solid #2196f3; padding: 15px; margin: 10px 0; color: #0d47a1;">
  <strong>核心概念：GET vs POST</strong><br>
  <strong>GET</strong>：数据附加在 URL 后（可见），长度受限，仅用于查询（如搜索框）。<br>
  <strong>POST</strong>：数据包含在 HTTP 请求体中（不可见），更安全，无长度限制，用于提交敏感数据（如密码）。
</div>
## 创建表单

使用 `<form>` 标签定义表单域，它不仅是控件的容器，还负责控制数据的提交方式。

- **`action`**：指定表单提交的目标 URL（处理数据的后端接口）。
- **`method`**：提交方式，默认为 `GET`，通常需手动设置为 `POST`。
- **`enctype`**：当表单包含文件上传（`type="file"`）时，必须设置为 `multipart/form-data`。

## 输入控件

`<input>` 是最通用的表单元素，通过 `type` 属性变换形态。

- **通用属性**：
    - `name`：**必填**，数据的“键”名，服务器通过此名称获取值。
    - `placeholder`：输入框内的灰色提示文字。
    - `required`：标记为必填项，浏览器会进行基础验证。
    - `value`：元素的默认值。
- **HTML5 新增类型**：提供了原生验证和针对移动端的键盘优化（如 `type="email"` 在手机上会弹出带 @ 的键盘）。

## 标签与辅助

`<label>` 标签用于定义控件的说明文本。

- **绑定机制**：通过 `for` 属性与控件的 `id` 绑定。
- **可用性提升**：用户点击 `<label>` 文本时，浏览器会自动聚焦或选中对应的控件。这对于复选框和单选框尤其重要，扩大了点击区域。

## 选择与多行文本

- **单选框 (`radio`)**：必须将一组单选框的 `name` 属性设为相同值，才能实现“多选一”的互斥效果。
- **下拉菜单 (`select`)**：`<select>` 定义容器，`<option>` 定义选项。
- **文本域 (`textarea`)**：用于多行文本输入，使用 `rows` 控制高度。
